C114 · Nivel 8

Boxplot

Nivel 3 · Distribución — visualiza medianas, cuartiles y valores atípicos (outliers) de conjuntos de observaciones.

V1 · Boxplot básico

Un solo grupo de datos mostrando mediana, Q1, Q3, whiskers y outliers. Ideal para inspeccionar la distribución de un endpoint.

Estructura de datos

// Array de arrays: cada sub-array es un grupo de observaciones
data: [
  [120, 132, 101, 134, 90, 230, 145, 160, 110, 98,
   155, 180, 125, 140, 115, 170, 88, 142, 165, 195]
]

// ECharts calcula automáticamente:
// min, Q1, mediana, Q3, max y outliers

V2 · Comparativo

Cuatro endpoints lado a lado para comparar distribuciones de tiempos de respuesta.

Estructura de datos

groups: ['GET /users', 'POST /orders', 'GET /products', 'PUT /settings'],
data: [
  [120, 132, 101, 134, 90, 230, ...],   // GET /users — 20 obs
  [220, 282, 201, 234, 290, 430, ...],   // POST /orders — 20 obs
  [80, 92, 71, 84, 65, 130, ...],        // GET /products — 20 obs
  [150, 162, 140, 195, 130, 280, ...],   // PUT /settings — 20 obs
]
Endpoint Observaciones (ms)

V3 · Horizontal

Ejes invertidos: categorías en Y, valores en X. Útil cuando los nombres de endpoints son largos.

Estructura de datos

// Mismos datos que V2, pero con ejes invertidos
// yAxis: { type: 'category' }  →  labels en el eje vertical
// xAxis: { type: 'value' }     →  tiempos en el eje horizontal
// grid: { left: 120 }          →  espacio para labels largos

V4 · Con scatter superpuesto

Cajas + puntos individuales de cada observación con jitter horizontal para evitar solapamiento.

Estructura de datos

// Combina boxplot + scatter con jitter
// Boxplot: dataset transform calcula cuartiles automáticamente
// Scatter: cada observación se dibuja con un desplazamiento
//          aleatorio en X para no solaparse

// jitter: Math.random() * 0.4 - 0.2  →  ±0.2 de offset
