Guía de desarrollo LayoutOS

Este documento describe las convenciones y patrones utilizados en el proyecto LayoutOS para crear interfaces consistentes y mantenibles.

Stack tecnológico

El proyecto utiliza Tailwind CSS v4 para estilos y Alpine.js 3 para interactividad. No se requiere bundler, todos los componentes funcionan directamente en el navegador.

Tailwind CSS

Se utilizan tokens de diseño predefinidos para mantener consistencia visual. Los componentes comparten clases base como bg-white dark:bg-zinc-900 border border-zinc-200 dark:border-zinc-800 rounded-xl shadow-sm para cards.

Alpine.js

Cada página define una función de datos que encapsula todo el estado y la lógica. Se evitan dependencias externas y se prioriza la simplicidad.

Estructura de archivos

  • src/pages/ — Páginas organizadas por nivel de complejidad
  • src/components/ — Componentes reutilizables
  • public/css/ — Estilos compilados

Convenciones de código

Cada archivo HTML es autocontenido. Las funciones Alpine se definen en un bloque <script> al final del body, antes de la carga de Alpine.
